# Политика безопасности

## Сообщение об уязвимости

Не публикуйте уязвимости, пароли, токены или ключи в issues, pull request или истории коммитов.

Для приватного проекта сначала сообщайте о проблеме владельцу репозитория через закрытый канал. В сообщении укажите краткое описание, возможный эффект и безопасный способ воспроизведения без секретных данных.

## Правила проекта

- Секреты передаются через безопасное хранилище GitHub или систему управления секретами, а не через исходный код.
- Реальные файлы `.env`, ключи и сертификаты запрещено добавлять в Git.
- Изменения в основной ветке проходят через pull request и проверку владельцем кода.
- При случайной публикации секрета его нужно немедленно отозвать и заменить: удаления файла из нового коммита недостаточно.
