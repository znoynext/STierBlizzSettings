# Profile authoring

1. Save supported current settings as a personal profile.
2. Export the profile and validate it by importing it in a test character.
3. Record client build, test date, selected modules and changes.
4. Obtain and record explicit permission before using a real player name, logos, images or settings.
5. Add the verified badge only after permission and live-client validation.
6. Re-test after client patches; mark incompatible or outdated profiles instead of silently applying them.

Profile IDs must contain only letters, digits, `_`, `-` or `.`, and be at most 128 bytes. Display names are required, limited to 80 bytes and cannot contain control characters; descriptions are limited to 1024 bytes. Graphics profiles must declare `unified` or `split` mode explicitly. Only documented registry keys are accepted, and imported metadata is always treated as untrusted display text.

The included **Example Competitive Tank** is fictional and deliberately unverified.
