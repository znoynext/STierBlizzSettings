# Manual live-client test plan

Run on a clean current Retail installation in English and Russian at 1080p, 1440p and above 1440p.

Focused 0.4 checks: validate all three presets; run the automatic 5-second post-apply comparison, standalone 20-second frame-time test and each current-vs-preset comparison; test Zone Graphics across world, dungeon, raid, PvP/arena and scenario/delve content; toggle the bottom FPS/ping indicator and verify it stops updating while disabled.

1. Open with `/stier`, `/stbs`, the minimap button and Blizzard Settings. Verify Graphics, UI Tweaks, Test FPS, Profiles and About appear on the left; Graphics Settings and Zone Graphics Switcher share a dedicated top bar above the Graphics page title and remain aligned without text overlap at every supported window size. The header must show the actually matching PRO, Optimized or Quality preset, then switch to Custom after one manual graphics change. Right-clicking the minimap button opens Profiles, `/stier zone` opens the nested Zone Graphics page, `/stier tweaks` opens UI Tweaks, `/stier fps` opens Test FPS and `/stier about` opens About.
   Verify navigation, presets, actions, graphics sub-tabs, the FPS modal Cancel button and the Blizzard Settings launcher share the flat Retail button style. Resize the window and confirm borders stay crisp; selected controls use a gold bar, hover/press transitions remain smooth, and only destructive Delete actions use a red fill.
2. Drag the minimap button around the minimap, reload the UI and verify its position persists.
3. Select PRO, Optimized and Quality in turn and verify expensive settings scale while projected textures, particles, outlines and texture resolution remain safe. Select unified mode, review the diff, cancel the popup, then confirm. Verify only base graphics and `RAIDsettingsEnabled=0` change. Repeat for split mode and verify `RAIDsettingsEnabled=1` plus raid values.
4. Confirm the apply popup names the change count, a backup is created before writes, unavailable values are skipped, and a failed write rolls every attempted setting back.
5. Verify the four-card Graphics dashboard updates live FPS, shows before/after/change after the five-second post-apply comparison, colors gains green and regressions red, and hides stale result values during a new measurement. In Test FPS, run one 20-second capture and verify the centered progress dialog briefly warns against camera movement, minimizing WoW and Alt+Tab; confirm it blocks mouse input, keeps the existing gold color, fills smoothly with the soft native texture/edge glow and Cancel immediately closes it. Verify average FPS, 1% Low, stability explanation, adaptive spike count, worst-frame time and advice; keep the scene fixed and compare the result with an external frame-time capture.
6. Run current-vs-PRO, current-vs-Optimized and current-vs-Quality. Each must capture 20 seconds before and after, replace raw result lines with four responsive comparison cards, show signed and color-coded average FPS, 1% Low and stability deltas, and place a small legend below the cards identifying left as the user's graphics and right as the tested preset. Retain exactly one rollback backup and restore every captured graphics value. Cancel during the current phase and candidate phase; in combat, verify restoration queues and Zone Graphics remains suspended until it completes.
7. Apply each built-in preset and verify a centered smooth 0–5 second progress dialog appears. When it finishes, confirm the dialog closes and Reload UI is offered automatically. Decline once and verify no persistent Reload UI button remains in Graphics; accept once and verify the successful reload path.
8. Use Undo graphics changes and verify the latest graphics backup is restored and a safety backup is created. Repeat after combat queuing.
9. Save more than five graphics profiles and backups. Verify Save Graphics accepts Enter, rejects whitespace-only names, shows success/failure, and all items remain selectable through scrolling. Apply/export/rename/delete profiles and restore/delete backups; verify every action reports a result.
10. Export and re-import graphics profiles; test profile mode and current mode. Reject interface-only, malformed, oversized, bad-checksum, duplicate-key, malformed-section, unsafe-metadata, unknown-CVar and future-schema imports.
11. Verify every Graphics Quality child, raid child, image AA, Multisample Alpha-Test, texture filtering and ray-traced shadows follows the profile. Verify monitor, display mode, resolution, Render Scale, UI scale, V-Sync, notch, latency, camera FOV, triple buffering, resampling, VRS, API/card, physics, FPS caps and color controls remain unchanged.
12. Test at 100%, 125% and 150% UI scale, minimum/default/maximum window sizes, long translated labels, larger fonts, window/page/status/hover animations, scroll/focus/escape behavior, header-only movement beyond every screen edge, resize persistence and the gold minimap icon.
13. On Profiles, verify the three internal views, select every backup after scrolling, then restore and delete it from the actions that reappear at the top.
14. Export `STBSA1`, open Import and verify it uses a large responsive multiline field instead of a popup. Paste it on a clean account, review the summary, confirm the backup-first warning, verify graphics/UI Tweaks/preferences/zone mappings/profiles, and confirm that empty/corrupted/oversized strings show an inline error while combat imports fail closed.
15. Verify About accurately describes the source audit, transactional backup, no telemetry, preserved controls and non-guaranteed FPS result.
16. Verify existing saved Interface & Gameplay data survives upgrades even though that user-facing section is hidden.
17. Test corrupted SavedVariables and a clean install; confirm there are no Lua errors, duplicated nested addon directories or stale `home` UI files.
18. Enable Zone Graphics and cross each supported content type. Confirm a backup is created only for a real diff, combat queues one validated operation, disabled mode never writes, and unknown instance types use the world mapping.
19. Enable the FPS/ping indicator. Compare Home/World values with the Blizzard micro-menu tooltip, verify the higher latency is shown, check independent color transitions, then disable it and confirm its ticker is cancelled.
20. Open UI Tweaks and verify the trust note, universal/optional grouping, flat dark checkboxes with bronze borders and gold checked states, smooth hover/press feedback, resample sharpness slider, maximum-camera-distance checkbox and hover tooltips. Confirm unavailable CVars are disabled. Toggle camera distance and verify `1.9 / 2.6` without accepting other values. Select the recommended `1 / 0.3 / 1` values, cancel Apply once, then confirm; verify one backup, readback verification, visible success and Undo with a safety backup. Give every tweak a distinctive value, then apply PRO, Optimized, Quality and a Zone Graphics change; confirm none of the six UI Tweaks changes.
21. Complete preset comparisons with faster, similar, mixed and slower synthetic/live outcomes. Verify the apply recommendation appears only after a successful restore and at least 5% gains in both Average FPS and 1% Low. Confirm the button names the tested preset, asks for confirmation, creates a new graphics backup, selects and applies that exact preset transactionally, starts the normal five-second measurement and disappears after acceptance.
22. Open every confirmation and short text-entry flow: Graphics apply/undo/reload, UI Tweaks apply/undo, tested-preset apply, profile apply/save/rename/import, backup restore/delete and profile delete. Verify all use the centered S-Tier modal with icon, dark surface, gold border, standard large fonts and shared buttons; destructive confirmations use red only for Accept. Enter validates required text and Escape closes cleanly. Confirm no generic `StaticPopup` remains.
