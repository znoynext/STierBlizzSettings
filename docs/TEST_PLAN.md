# Manual live-client test plan

Run on a clean current Retail installation in English and Russian at 1080p, 1440p and above 1440p.

Focused 0.4 checks: validate all three presets; run the automatic 5-second post-apply comparison for an immediate apply, verify a combat-queued apply skips stale comparison, run the standalone 20-second frame-time test and each current-vs-preset comparison; test Zone Graphics across world, dungeon, raid, PvP/arena and scenario/delve content; toggle the bottom FPS/ping indicator and verify it stops updating while disabled.

The current UI has no unified/split, lighter-raid, or quick/accurate benchmark selector. Split profiles remain clearly labelled compatibility state with unified-copy conversion and a secondary advanced apply; benchmark mode is no longer persisted or exported, while old version-1 bundles containing a valid value remain import-compatible and ignore it.

1. Open with `/stier`, `/stbs`, the minimap button and Blizzard Settings. Verify Graphics, UI Tweaks, Test FPS, Profiles and About appear on the left; Graphics Settings and Zone Graphics Switcher share a dedicated top bar above the Graphics page title and remain aligned without text overlap at every supported window size. The header must show the actually matching PRO, Optimized or Quality preset, then switch to Custom after one manual graphics change. Right-clicking the minimap button opens Profiles, `/stier zone` opens the nested Zone Graphics page, `/stier tweaks` opens UI Tweaks, `/stier fps` opens Test FPS and `/stier about` opens About.
   Verify navigation, presets, actions, graphics sub-tabs, the FPS modal Cancel button and the Blizzard Settings launcher share the flat Retail button style. Resize the window and confirm borders stay crisp; selected controls use a gold bar, hover/press transitions remain smooth, and only destructive Delete actions use a red fill.
2. Drag the minimap button around the minimap, reload the UI and verify its position persists.
3. Select PRO, Optimized and Quality in turn and verify expensive settings scale while projected textures, particles, outlines and texture resolution remain safe. Before Apply, confirm the choice changes only the visible draft and not persisted applied preset/mode. Confirm there is no mode or lighter-raid toggle. Review the diff, cancel the dialog, then confirm; verify the built-in flow applies only the active base set with `RAIDsettingsEnabled=0` and commits applied state only after success. Separately add a valid personal profile whose stored mode is split. Verify the Profiles list and preview label it as legacy; conversion requires confirmation/name, creates a new valid unified copy, uses its base values everywhere, retains its raid/inactive-raid data, preserves the original, and does not change active CVars. Then use the secondary advanced action and verify its warning, backup, `RAIDsettingsEnabled=1`, retained raid values, transaction feedback and queued-completion behavior. A malformed split fixture must expose neither a successful conversion nor any writes.
4. Confirm the apply dialog names the actual change count and a backup is created before real writes. Reapply an already-active built-in preset, personal profile and UI Tweaks selection; verify `unchanged`, zero writes, no backup/history growth, no FPS measurement or Reload prompt, and clear already-active feedback. Repeat an already-matching Backup History restore and verify no safety backup is created. Force the second read of one changed CVar to fail and verify `snapshot-failed`, zero writes and no new backup. Keep another submitted setting unavailable while a safe changed setting remains readable and verify only the safe target applies. For multiple writes, fail a middle write and confirm reverse rollback uses the exact pre-write snapshot rather than later backup/post-failure reads; force rollback read-back failure and confirm `rollback-failed` is reported. Identical, skipped, unsupported and unavailable entries must remain visible in result counts.
5. Verify the four-card Graphics dashboard updates live FPS, shows before/after/change after the five-second post-apply comparison, colors gains green and regressions red, and hides stale result values during a new measurement. In Test FPS, run one 20-second capture and verify the centered progress dialog briefly warns against camera movement, minimizing WoW and Alt+Tab; confirm it blocks mouse input, keeps the existing gold color, fills smoothly with the soft native texture/edge glow and Cancel immediately closes it. Verify average FPS, 1% Low, stability explanation, adaptive spike count, worst-frame time and advice; keep the scene fixed and compare the result with an external frame-time capture.
6. Run current-vs-PRO, current-vs-Optimized and current-vs-Quality. Each must capture 20 seconds before and after, replace raw result lines with four responsive comparison cards, show signed and color-coded average FPS, 1% Low and stability deltas, and place a small legend below the cards identifying left as the user's graphics and right as the tested preset. Snapshot Backup History before each run and confirm it is structurally unchanged after successful verified restore, including with `backupLimit=1`. Cancel during the current phase and candidate phase and confirm the same cleanup. In combat, verify restoration queues, its session backup remains until actual post-combat success, and Zone Graphics remains suspended until completion. With `backupLimit=1`, force restore and rollback failures, reinitialize SavedVariables, and confirm the pre-comparison and safety records of only that session remain while user/manual/zone records and another synthetic session are untouched.
7. Apply each built-in preset outside combat and verify a centered smooth 0–5 second progress dialog appears. When it finishes, confirm the dialog closes and Reload UI is offered automatically. Queue one apply in combat, move to a visibly different scene, then leave combat: verify settings apply and persisted applied preset/mode commit only now; no automatic post measurement starts, the UI explains that the stale comparison was skipped and Test FPS remains available, and Reload UI is still offered. Repeat with cancellation and forced `failed`, `rolled-back`, and `rollback-failed` results: verify Pending clears where applicable, persisted applied state remains unchanged, a clear final error appears, and neither measurement nor Reload UI starts. Decline Reload once and verify no persistent Reload UI button remains in Graphics; accept once and verify the successful reload path.
8. Use Undo graphics changes after a built-in preset, personal profile, imported profile, manual Zone Graphics and full-addon import; verify the latest retained user-change backup is restored by stable ID and a safety backup is created for a real restore. Insert automatic Zone Graphics, FPS-comparison temporary, restore-safety and multiple automatic records after the user backup and confirm Undo still chooses the user backup. Confirm `legacy`, unknown and manual-snapshot sources are not inferred as user changes. Delete or expire the candidate after opening confirmation, insert another eligible backup, then accept: the action must fail for the captured ID rather than restoring the newer record. Separately verify Backup History can explicitly restore any selected retained graphics backup. With a controlled legacy fixture, verify complete, partial and all-unavailable restore feedback shows restored/identical/skipped/unavailable/failed counts; removed or unavailable settings must not block compatible safe values, while write failure remains failed and rollback failure is reported separately. Queue Backup History Restore and Undo in combat and confirm both show a neutral pending message rather than failure or completion; after combat, only centralized completion may report complete, partial, unavailable, rolled-back or rollback-failed. At `backupLimit=1`, force an ordinary restore failure and confirm the selected source and safety backup both survive normalization for retry.
9. Save more than five graphics profiles and backups. Before two saves, deliberately make persisted `graphicsMode` disagree with the live Blizzard `RAIDsettingsEnabled` in both directions; verify the saved profile follows the live value and Save performs no CVar writes. Make the live CVar unavailable, then invalid, and confirm both saves fail without allocating/persisting a profile. Also verify Save Graphics accepts Enter, rejects whitespace-only names, shows success/failure, and all items remain selectable through scrolling. Select a backup, open Restore/Delete confirmation, create another backup before accepting, and verify the originally selected stable ID is still the only target. Apply/export/rename/delete profiles and restore/delete backups; verify every action reports a result.
10. Export graphics profiles and verify the deterministic `STBS1` payload. There is no current Profiles action that opens individual `STBS1` import; verify that the visible Import / Export view exposes only full-addon import. Run `lua Tests/run.lua` for the existing parser, checksum, schema, allowlist, and malformed/oversized-input coverage. If the internal import flow is invoked diagnostically, also test profile mode, current mode, and rejection of interface-only input by the graphics-specific UI flow.
11. Verify every active base Graphics Quality child, image AA, Multisample Alpha-Test, texture filtering and ray-traced shadows follows each built-in profile. Verify retained raid children only through the separate legacy split-profile compatibility check from step 3. Verify monitor, display mode, resolution, Render Scale, UI scale, V-Sync, notch, latency, camera FOV, triple buffering, resampling, VRS, API/card, physics, FPS caps and color controls remain unchanged.
12. Test at 100%, 125% and 150% UI scale, minimum/default/maximum window sizes, long translated labels, larger fonts, window/page/status/hover animations, scroll/focus/escape behavior, header-only movement beyond every screen edge, resize persistence and the gold minimap icon.
13. On Profiles, verify the three internal views, select every backup after scrolling, then restore and delete it from the actions that reappear at the top.
14. Export `STBSA1`, open Import and verify it uses a large responsive multiline field instead of a popup. Paste it on a clean account, review the summary, confirm the backup-first warning, and verify the exact shared fields: graphics, verified applied preset/mode (including Custom), available UI Tweaks, performance-widget enabled state, Zone Graphics enabled/assignments, and personal profiles. New exports must not contain `benchmarkMode`; an old version-1 bundle with `quick` or `accurate` must import successfully without persisting the field, while another value is rejected. Backup/transaction history, runtime draft and screen positions must not be exported. Before import, set distinctive target-account values for backup limit, window size, Performance Widget position and minimap angle plus one synthetic unknown local preference; confirm every one survives while shared fields and imported profiles update. Confirm that empty/corrupted/oversized strings show an inline error while combat and future-schema imports fail closed without mutating SavedVariables.
15. Verify About accurately describes the source audit, transactional backup, no telemetry, preserved controls and non-guaranteed FPS result.
16. Verify existing saved Interface & Gameplay data survives upgrades even though that user-facing section is hidden.
17. Test a clean database, missing/corrupted version metadata, sequential schema `2 -> 3 -> 4 -> 5 -> 6`, direct migrations from schemas 3, 4 and 5, repeated schema-6 initialization and a synthetic future schema. Confirm schema `5 -> 6` removes only legacy `benchmarkMode`, while valid preferences, profiles, backups, transactions, Zone Graphics assignments and device-local positions survive; actual graphics detection runs once. Every legacy backup must receive one stable unique ID and normalized source that survive reinitialization, insertion, reordering and retention. Verify known historical triggers map to the documented source, unknown triggers/invalid sources become `legacy`, and each current workflow creates the expected source without changing its diagnostic trigger. For the future schema, attempt managed settings, applied-state commit, backup create/finalize/restore/delete, profile save/rename/delete, Zone Graphics and representative local preferences, full import and logging. Every persistent account mutation must return `database-schema-unsupported`; the original database and runtime fallback must remain structurally untouched, while current-schema persistence still succeeds. Also confirm there are no Lua errors, duplicated nested addon directories or stale `home` UI files.
18. Enable Zone Graphics and cross each supported content type. Confirm a backup is created only for a real diff, combat queues one validated operation, disabled mode never writes, and unknown instance types use the world mapping.
19. Enable the FPS/ping indicator. Compare Home/World values with the Blizzard micro-menu tooltip, verify the higher latency is shown, check independent color transitions, then disable it and confirm its ticker is cancelled.
20. Open UI Tweaks and verify the trust note, universal/optional grouping, flat dark checkboxes with bronze borders and gold checked states, smooth hover/press feedback, resample sharpness slider, maximum-camera-distance checkbox and hover tooltips. Confirm unavailable CVars are disabled. Toggle camera distance and verify `1.9 / 2.6` without accepting other values. Select the recommended `1 / 0.3 / 1` values, cancel Apply once, then confirm; verify one backup, readback verification, visible success and Undo with a safety backup. Give every tweak a distinctive value, then apply PRO, Optimized, Quality and a Zone Graphics change; confirm none of the six UI Tweaks changes.
21. Complete preset comparisons with faster, similar, mixed and slower synthetic/live outcomes. Verify the apply recommendation appears only after a successful restore and at least 5% gains in both Average FPS and 1% Low. Confirm the button names the tested preset, asks for confirmation, creates a new graphics backup, selects and applies that exact preset transactionally, starts the normal five-second measurement and disappears after acceptance.
22. In combat, queue and complete one operation of each type: Graphics, automatic Zone Graphics, manual Zone Graphics, UI Tweaks, backup recovery and FPS comparison recovery. For applied/no-op/failure/rollback/rollback-failure outcomes, verify the relevant page leaves Pending state and shows its final status; Zone Graphics updates only verified active state, UI Tweaks clears its draft only on success, and recovery reports whether restoration succeeded.
23. Open every currently reachable confirmation and short text-entry flow: Graphics apply/undo/reload, UI Tweaks apply/undo, tested-preset apply, profile apply/save/rename, backup restore/delete and profile delete. Verify all use the centered S-Tier modal with icon, dark surface, gold border, standard large fonts and shared buttons; destructive confirmations use red only for Accept. Enter validates required text and Escape closes cleanly. Confirm no generic `StaticPopup` remains. The unreachable individual profile-import dialog is covered as source/test compatibility, not as current navigation.
